# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Obsidian native (PluginSettingTab, Setting, Menu, StatusBarItem) |
| Icon library | none (D-01: text badge style, no emoji/icons) |
| Font | Obsidian theme default (inherited, not customized) |

**Note:** This is an Obsidian plugin. All UI surfaces use Obsidian's built-in component APIs, which inherit the user's active Obsidian theme (light/dark, custom CSS). The plugin does NOT inject custom fonts, custom CSS frameworks, or external component libraries. Custom CSS is limited to a single `styles.css` for plugin-specific overrides only.

---

## Spacing Scale

Obsidian's native Setting components handle all spacing internally. The plugin does not define custom spacing tokens in Phase 1.

Declared values for the single custom CSS element (status bar popover styling, if needed):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding within status bar text |
| sm | 8px | Padding inside popover menu items |
| md | 16px | Not used in Phase 1 |
| lg | 24px | Not used in Phase 1 |
| xl | 32px | Not used in Phase 1 |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Status bar item uses Obsidian's default status bar padding (inherited from theme). No custom spacing overrides are applied to the status bar element.

---

## Typography

All text rendering uses Obsidian's theme typography. The plugin does not declare custom font sizes or weights. The following documents the semantic roles used:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Status bar text | Obsidian theme `--status-bar-font-size` (typically 12px) | 400 (regular) | 1.5 |
| Settings heading | Obsidian theme `--h3-size` (typically 16-18px) | 600 (semibold) | 1.2 |
| Settings label | Obsidian theme `--font-ui-small` (typically 14px) | 400 (regular) | 1.5 |
| Settings description | Obsidian theme `--font-ui-smaller` (typically 12-13px) | 400 (regular) | 1.5 |

**Constraint:** The plugin MUST NOT override Obsidian theme typography. All font rendering is inherited.

---

## Color

All colors are inherited from the Obsidian theme. The plugin does not inject custom color tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-primary)` | Obsidian app background (inherited) |
| Secondary (30%) | `var(--background-secondary)` | Settings tab container, popover background (inherited) |
| Accent (10%) | `var(--interactive-accent)` | Not used in Phase 1 (no primary CTAs, no accent elements) |
| Destructive | `var(--text-error)` | API key validation failure text only |

**Status bar state colors** (these are the only semantic color assignments the plugin makes):

| State | CSS class | Text color | Source |
|-------|-----------|------------|--------|
| `AI: idle` | `.canvas-ai-status--idle` | `var(--text-muted)` | Unobtrusive when idle |
| `AI: thinking` | `.canvas-ai-status--thinking` | `var(--text-normal)` | Normal emphasis when active |
| `AI: error` | `.canvas-ai-status--error` | `var(--text-error)` | Error semantic color |
| `AI: no API key` | `.canvas-ai-status--error` | `var(--text-error)` | Same as error state |

Accent reserved for: Nothing in Phase 1. Accent color usage begins in Phase 3 when generated nodes need visual distinction.

## Component Inventory

Phase 1 has exactly 4 UI surfaces. Each is documented below with its states, interactions, and exact copy.

### 1. Status Bar Indicator

**Obsidian API:** `Plugin.addStatusBarItem()` returns `HTMLElement`

**States:**

| State | Text Content | CSS Class | Visible? | Source |
|-------|-------------|-----------|----------|--------|
| Idle | `AI: idle` | `canvas-ai-status--idle` | Yes (when enabled canvas is active) | D-01 |
| Thinking | `AI: thinking` | `canvas-ai-status--thinking` | Yes | D-01, D-03 |
| Error | `AI: error` | `canvas-ai-status--error` | Yes | D-01 |
| No API key | `AI: no API key` | `canvas-ai-status--error` | Yes | D-15 |
| Canvas disabled | (hidden) | -- | No (`display: none`) | D-04 |
| No canvas active | (hidden) | -- | No (`display: none`) | Inferred |

**Interactions:**

| Trigger | Action | Source |
|---------|--------|--------|
| Click status bar item | Open status popover (Menu) at click position | D-02 |
| Canvas becomes active + enabled | Show status bar, set to current state | D-04, inferred |
| Canvas becomes active + disabled | Hide status bar | D-04 |
| Leave canvas view | Hide status bar | Inferred |
| Debounce fires | Set text to `AI: thinking` | D-03 |
| Debounce completes (Phase 1: immediately, since no generation yet) | Set text to `AI: idle` | D-03 |

**CSS (styles.css):**

```css
.canvas-ai-status {
  cursor: pointer;
}
.canvas-ai-status--idle {
  color: var(--text-muted);
}
.canvas-ai-status--thinking {
  color: var(--text-normal);
}
.canvas-ai-status--error {
  color: var(--text-error);
}
```

### 2. Status Popover

**Obsidian API:** `Menu` class with disabled (display-only) items

**Content (always exactly 2 items):**

| Item | Format | Example | Disabled? |
|------|--------|---------|-----------|
| Current state | `State: {state}` | `State: idle` | Yes (display only) |
| Last trigger | `Last trigger: {time}` or `Last trigger: never` | `Last trigger: 2:34:15 PM` | Yes (display only) |

**Time format:** Use `Date.toLocaleTimeString()` with no arguments (user's locale default). If no trigger has occurred since plugin load, display `never`.

**Interactions:**

| Trigger | Action |
|---------|--------|
| Click status bar | `menu.showAtMouseEvent(evt)` |
| Click outside popover | Menu closes (built-in Menu behavior) |
| Press Escape | Menu closes (built-in Menu behavior) |

### 3. Settings Tab

**Obsidian API:** `PluginSettingTab` with `Setting` components

**Tab name:** `Canvas AI`

**Layout: 2 sections with headings (D-05)**

#### Section 1: API Keys

| Setting | Component | Placeholder | Description | Validation |
|---------|-----------|-------------|-------------|------------|
| Claude API key | `Setting.addText()` | `sk-ant-...` | `Your Anthropic API key for Claude Opus 4.6` | Format check on blur: non-empty + starts with `sk-ant-`. Inline result below field. |
| Runware API key | `Setting.addText()` | `rw-...` | `Your Runware API key for image generation` | Format check on blur: non-empty. Inline result below field. |

**Validation feedback (D-06, adapted per RESEARCH.md Pitfall 6):**

| Result | Display | Method |
|--------|---------|--------|
| Valid format | Green text: `Format OK — will verify on first use` | `descEl.setText()` + `descEl.addClass('canvas-ai-valid')` |
| Invalid format | Red text: `Invalid key format. Expected: sk-ant-...` | `descEl.setText()` + `descEl.addClass('canvas-ai-invalid')` |
| Empty | Gray text (original description) | No validation indicator |

**Validation CSS:**

```css
.canvas-ai-valid {
  color: var(--text-success);
}
.canvas-ai-invalid {
  color: var(--text-error);
}
```

**API key display:** Plain text, not masked (D-07).

#### Section 2: Behavior

| Setting | Component | Config | Description |
|---------|-----------|--------|-------------|
| Debounce delay | `Setting.addSlider()` | `setLimits(1, 10, 1)`, `setDynamicTooltip()` | `Seconds of idle before AI generates` |
| Debug mode | `Setting.addToggle()` | Default: `false` | `Log all canvas events to developer console` |

**Slider behavior (D-08):**
- Range: 1 to 10 seconds, step 1
- Dynamic tooltip shows current integer value while dragging
- Default: 3 seconds
- Value persists immediately on change via `saveSettings()`

**Toggle behavior (D-14):**
- Default: off
- When on: all detected canvas events (create, edit, move, delete) logged to `console.log` with event type and timestamp
- No visual indicator beyond the toggle state itself

### 4. Canvas Context Menu Item

**Obsidian API:** `workspace.on('canvas:node-menu')` + `Menu.addItem()`

**Menu item:**

| Condition | Menu item title | Section |
|-----------|----------------|---------|
| Canvas AI is enabled for this canvas | `Disable Canvas AI` | `canvas-ai` |
| Canvas AI is disabled for this canvas | `Enable Canvas AI` | `canvas-ai` |

**Interactions:**

| Trigger | Action |
|---------|--------|
| Click "Disable Canvas AI" | Add canvas file path to `disabledCanvases` array, save settings, hide status bar | 
| Click "Enable Canvas AI" | Remove canvas file path from `disabledCanvases` array, save settings, show status bar |

**Separator:** `menu.addSeparator()` before the Canvas AI item to visually separate from Obsidian's native menu items.

## Interaction Contracts

### First-Run Experience

When the plugin loads for the first time (no API keys configured):

1. Status bar shows `AI: no API key` in error color
2. One-time dismissable `Notice` appears: `Canvas AI: Set up your API keys in Settings → Canvas AI` (D-15)
3. Notice uses Obsidian's built-in `Notice` class with default timeout (4 seconds)
4. Notice is shown only ONCE per plugin load session (track with a boolean flag, not persisted)

### State Transitions

```
Plugin loads
  ├─ No API key → "AI: no API key" + Notice (once)
  ├─ Has API key + canvas active + enabled → "AI: idle"
  ├─ Has API key + canvas active + disabled → (hidden)
  └─ Has API key + no canvas active → (hidden)

Canvas event detected (create/edit/move/delete)
  └─ Debounce timer starts (internal, no UI change) → D-13

Debounce fires
  └─ "AI: idle" → "AI: thinking"

Generation completes (Phase 1: immediate since no actual generation)
  └─ "AI: thinking" → "AI: idle"

Error occurs
  └─ Any state → "AI: error"

User disables canvas (context menu)
  └─ Status bar hidden → D-04

User enables canvas (context menu)
  └─ Status bar shown with current state

User leaves canvas view
  └─ Status bar hidden

User opens/returns to enabled canvas
  └─ Status bar shown with current state
```

### Command Palette

| Command ID | Command Name | Condition | Action |
|------------|-------------|-----------|--------|
| `toggle-canvas-ai` | `Toggle Canvas AI for current canvas` | Only available when a canvas view is active | Toggle current canvas enabled/disabled state |
