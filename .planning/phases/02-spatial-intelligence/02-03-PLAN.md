Build the context builder that orchestrates all spatial modules into a complete SpatialContext, serializes a structured narrative for Claude, and creates the barrel export for the spatial module.

Purpose: This is the top of the spatial analysis pipeline. It takes raw CanvasNodeInfo[] and edges, runs proximity analysis, clustering, placement, and produces a structured narrative string that Phase 3 will inject into the Claude system prompt. It also filters nodes by relevance threshold (SPAT-07) to control API costs.

Output: Context builder with narrative serialization, node filtering, the buildSpatialContext orchestrator function, barrel export index.ts, and comprehensive tests.

    export interface RelevantNode {
      node: CanvasNodeInfo;

    **Functions to implement:**

    1. `describeDirection(from: CanvasNodeInfo, to: CanvasNodeInfo): string`
       - Per D-15: include spatial directions
       - Compute dx and dy between centers (using computeCenter)
       - Canvas Y increases DOWNWARD: negative dy = above, positive dy = below
       - Threshold of 30px for directional significance (below that = "nearby")
       - Parts array: if abs(dy) > 30, push 'above' or 'below'. If abs(dx) > 30, push 'to the left' or 'to the right'
       - Return parts.join(', ') or 'nearby' if empty

    2. `filterRelevantNodes(nodes: CanvasNodeInfo[], proximityGraph: ProximityGraph, triggerNodeId: string, threshold: number): { relevant: RelevantNode[]; excluded: CanvasNodeInfo[] }`
       - Per D-14 and SPAT-07: filter by relevance score threshold
       - For each node (excluding trigger node), find the ProximityPair that links it to the trigger node
       - If relevance >= threshold, include in relevant array with direction from trigger
       - The trigger node itself is always included (it's the anchor)
       - Sort relevant nodes by relevance descending (most relevant first)
       - Return both relevant and excluded lists

    3. `serializeNarrative(focusCluster: ClusterInfo | null, relevantNodes: RelevantNode[], outliers: OutlierNode[], triggerNode: CanvasNodeInfo): string`
       - Per D-13: structured narrative format with natural language
       - Structure:
         ```
         ## Canvas Context

         The user just edited a node. Here is the spatial context of their canvas:

         ### Active Node
         - [type] "content truncated to 100 chars"

         ### Focus Area (N nodes)    [if focusCluster exists]
         - [type] "content" (direction)
         ...

         ### Nearby Context (N nodes)  [if relevantNodes.length > 0]
         - [type] "content" (direction, relevance: 0.XXX)
         ...

         ### Peripheral (N distant nodes)  [if outliers.length > 0, per D-16]
         - "content truncated to 50 chars" (far direction)
         ...
         ```
       - Per D-16: outlier nodes briefly mentioned as peripheral context
       - Truncate content: 100 chars for focus/relevant, 50 chars for peripheral
       - Use `truncate(text: string, maxLen: number): string` helper (slice + '...' if needed)

    4. `buildSpatialContext(nodes: CanvasNodeInfo[], edges: CanvasEdgeInfo[], triggerNodeId: string, config?: Partial<SpatialConfig>): SpatialContext`
       - This is the top-level orchestrator function that Phase 3 will call
       - Merge config with DEFAULT_SPATIAL_CONFIG
       - Steps:
         1. Build proximity graph: `buildProximityGraph(nodes, edges, mergedConfig)`
         2. Compute adaptive epsilon: `computeAdaptiveEpsilon(nodes, mergedConfig.adaptiveMultiplier)`
         3. Run DBSCAN: `dbscan(nodes, epsilon, mergedConfig.minPoints)`
         4. Build ClusterInfo[] from ClusterResult (compute centroids, attach node data)
         5. Find focus cluster: `findFocusCluster(clusterInfos, triggerNodeId, nodes)`
         6. Filter relevant nodes: `filterRelevantNodes(nodes, graph, triggerNodeId, mergedConfig.relevanceThreshold)`
         7. Build outlier list from DBSCAN noise nodes
         8. Compute placement suggestions: `computeOrbitalPlacements(triggerNode, 3, { width: 300, height: 200 }, nodes, mergedConfig.placementGap)` -- default 3 suggestion positions with standard node size
         9. Serialize narrative: `serializeNarrative(focusCluster, relevantFiltered, outlierInfos, triggerNode)`
         10. Return complete SpatialContext

       - Edge cases:
         - If triggerNodeId not found in nodes, use the first node as trigger (or return empty context if no nodes)
         - If no nodes at all, return SpatialContext with null focus cluster, empty arrays, minimal narrative, empty placements
