       - Wraps in try/catch silently (node may have been destroyed)

    3. `addNodeCssClass(node: any, className: string): void`
       - Adds CSS class to `node.nodeEl` if it exists: `node.nodeEl?.addClass(className)`
       - Per Pitfall 8: also set a data attribute as persistent marker: `node.unknownData = { ...node.unknownData, canvasAiStreaming: true }` (for re-application on re-render)

    4. `removeNodeCssClass(node: any, className: string): void`
       - Removes CSS class from `node.nodeEl`: `node.nodeEl?.removeClass(className)`
       - Clear the data attribute: delete `node.unknownData?.canvasAiStreaming`

    5. `resizeNode(node: any, height: number): void`
       - Calls `node.moveAndResize({ x: node.x, y: node.y, width: node.width, height })` to resize after streaming completes
       - Wraps in try/catch

    6. `requestCanvasSave(canvas: any): void`
       - Calls `canvas.requestSave()` to persist changes after streaming completes
       - Wraps in try/catch

    Add JSDoc comments referencing FOUN-13 warning: these write methods use internal API only, never vault.modify(), because canvas requestSave would overwrite file changes.
