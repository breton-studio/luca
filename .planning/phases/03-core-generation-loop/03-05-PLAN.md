  <what-built>
    Complete Phase 3 generation pipeline: Canvas idle trigger -> spatial context -> Claude API streaming -> pre-allocated text nodes with progressive rendering -> token budget tracking -> taste profile integration -> error handling with retry.
  </what-built>
  <how-to-verify>
    **Prerequisites:**
    - Ensure the plugin is loaded in your Obsidian vault (copy main.js, manifest.json, styles.css to `.obsidian/plugins/canvas-ai/`)
    - Have a valid Claude API key entered in Settings > Canvas AI

    **Test 1: Basic generation**
    1. Open a canvas in Obsidian
    2. Create a text node and type some content (e.g., "Design principles for a productivity app")
    3. Wait ~3 seconds (debounce delay)
    4. Observe: Status bar should show "AI: thinking" then "AI: streaming"
    5. A new node should appear near your node with AI-generated content
    6. The new node should have a different color (purple/preset 6 by default)
    7. Status bar should return to "AI: idle"

    **Test 2: Multi-node generation**
    1. Create several nodes with rich content (3-4 nodes in a cluster about a topic)
    2. Wait for generation
    3. Claude may generate 1-3 nodes depending on context richness
    4. If multiple nodes appear, they should appear sequentially (not all at once)

    **Test 3: Streaming animation**
    1. Create or edit a node
    2. Watch for the pre-allocated empty node to appear with a pulsing border
    3. The pulsing should stop when text starts appearing
    4. Text should stream in progressively (not appear all at once)

    **Test 4: Settings sections**
    1. Open Settings > Canvas AI
    2. Verify 5 sections: API Keys, Behavior, Token Budget, AI Node Appearance, Taste Profile
    3. Try adjusting the daily token limit slider (100K-2M range)
    4. Try changing the node color dropdown
    5. Click "Edit profile" to verify it opens/references the taste profile

    **Test 5: Token budget**
    1. Check Today's usage in settings (should show tokens used from Test 1-2)
    2. Set the daily token limit very low (e.g., 100K)
    3. If budget is exceeded, status bar should show "AI: budget"
    4. A notice should appear explaining the pause
    5. Toggle "Override budget" in settings -- generation should resume
  </how-to-verify>
  <resume-signal>Type "approved" if the generation loop works, or describe any issues you observe</resume-signal>
