# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Core Generation Loop. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none -- Obsidian plugin, not a web app |
| Preset | not applicable |
| Component library | Obsidian Plugin API (`Setting`, `Notice`, `Menu`) |
| Icon library | none -- text-only status indicators per Phase 1 D-01 |
| Font | Obsidian theme font (inherited via `var(--font-text)`) |

**Rationale:** This is an Obsidian plugin. All UI is built with Obsidian's Setting API, vanilla DOM manipulation, and CSS using Obsidian's CSS custom properties. No external component library, no Tailwind, no shadcn. Styles live in `styles.css` at the project root.

|------|------|--------|-------------|
| Body | `var(--font-text-size)` (Obsidian default, typically 16px) | 400 (regular) | 1.5 |
| Label | `var(--font-smallest)` (Obsidian default, typically 12px) | 400 (regular) | 1.4 |

## Color

This plugin uses Obsidian's CSS custom properties exclusively. The 60/30/10 split maps to Obsidian's existing theme structure:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-primary)` | Canvas background, settings background (inherited) |
| Secondary (30%) | `var(--background-secondary)` | Settings tab sidebar, status bar background (inherited) |
| Accent (10%) | `var(--interactive-accent)` | Budget override button only, pulsing border on streaming nodes |
| Destructive | `var(--text-error)` | Budget exceeded status bar text, API error states |
| Success | `var(--text-success)` | Budget usage below 80% indicator |
| Muted | `var(--text-muted)` | Status bar idle state, secondary labels in settings |

Accent reserved for:
- The "Continue anyway" budget override button background
- The pulsing border animation on pre-allocated nodes while waiting for the first token

### AI Node Color (D-05, D-06, D-07)

AI-generated canvas nodes use Obsidian's built-in canvas node color system. Canvas nodes support a `color` property with values `"1"` through `"6"` mapping to Obsidian's preset palette, or a hex string for custom colors.

| Property | Default Value | Configurable |
|----------|---------------|-------------|
| AI node color | `"6"` (canvas preset color 6 -- typically purple/violet in default themes) | Yes, via settings |
| AI node border during streaming | `var(--interactive-accent)` with pulsing animation | No |

**Rationale for color "6" default:** Obsidian canvas color presets are theme-adaptive. Using a preset number rather than a hardcoded hex ensures AI nodes look appropriate in any theme (light or dark). Color `"6"` is visually distinct from the default uncolored nodes while remaining subdued. The user can override this in settings (D-06).

**Why not literal inverse:** D-05 says "inverse of default node color." Since Obsidian canvas nodes can have any color or no color, computing a true inverse is fragile. Using a preset color number that is visually distinct from uncolored nodes achieves the same goal (immediate visual distinction) with theme safety.

### Settings Tab Extensions

New sections added below existing "Behavior" section in this order:

1. **Token Budget** (heading)
   - Daily token limit: slider, range 100K-2M tokens, step 100K, default 500K
   - Today's usage: read-only text display `{used} / {limit} tokens`
   - Override budget: toggle, default off, resets daily

2. **AI Node Appearance** (heading)
   - Node color: dropdown with options "1" through "6" plus "Custom hex", default "6"
   - If "Custom hex" selected: text input for hex value, validated with `#` prefix check

3. **Taste Profile** (heading)
   - Profile location: read-only text showing path
   - Edit profile: button that opens the taste profile file in Obsidian editor
   - Reset to default: button with undo-notice confirmation

### Canvas Node Streaming Animation

```css
/* Pulsing border for pre-allocated nodes awaiting first token (D-04) */
.canvas-ai-node--streaming {
  animation: canvas-ai-pulse 1.5s ease-in-out infinite;
}

@keyframes canvas-ai-pulse {
  0%, 100% {
    box-shadow: 0 0 0 2px var(--interactive-accent);
  }
  50% {
    box-shadow: 0 0 0 4px color-mix(in srgb, var(--interactive-accent) 40%, transparent);
  }
}

/* Remove animation once content starts streaming */
.canvas-ai-node--streaming.has-content {
  animation: none;
  box-shadow: none;
}
```

**Animation specification:**
- Duration: 1.5 seconds per cycle
- Easing: ease-in-out
- Effect: box-shadow pulsing from 2px to 4px spread, accent color fading to 40% opacity at peak
- Stops when: first token arrives (remove `.canvas-ai-node--streaming` class)
- Must not cause layout shift (box-shadow only, no border-width changes)

### Budget Exceeded Notice

- Type: Obsidian `Notice` (dismissable)
- Duration: 0 (persistent until dismissed)
- Content: heading + body + settings link (see copywriting contract)
- Shown: once when budget is first exceeded, not repeated until next day

## Interaction Contracts

### Generation Flow State Machine

```
IDLE --[canvas event]--> DEBOUNCING --[timeout]--> THINKING --[first token]--> STREAMING --[complete]--> IDLE
                                                       |                            |
                                                       v                            v
                                                   ERROR --[retry]--> THINKING   ERROR --[retry]--> STREAMING
                                                       |
                                                       v
                                                   IDLE (after max retries)

BUDGET_EXCEEDED interrupts at THINKING stage:
THINKING --[budget check fail]--> BUDGET --[override]--> THINKING
```

### Status Bar State Transitions

| From | To | Trigger |
|------|----|---------|
| idle | thinking | Debounce fires, generation begins |
| thinking | streaming | First token received from Claude API |
| streaming | idle | All nodes complete, generation finished |
| thinking | error | API call fails |
| streaming | error | Stream interrupted |
| error | idle | After retry backoff + successful retry, or max retries exhausted |
| thinking | budget | Daily token cap exceeded |
| budget | thinking | User enables "Override budget" toggle |
| any | idle | Plugin disabled or canvas switched |

### Node Streaming Sequence (D-01, D-02, D-03)

1. **Pre-allocate:** Create empty canvas node at computed placement position with AI node color and `canvas-ai-node--streaming` CSS class. Node dimensions: width 300px, height 200px (matches Phase 2 `computeOrbitalPlacements` default).
2. **Pulse:** Node border pulses while waiting for first token (D-04). Status bar shows `AI: thinking`.
3. **Stream:** Tokens arrive, buffered at 200-300ms intervals (GENP-04). Remove streaming class on first buffer flush. Status bar shows `AI: streaming`. Node content updates in place via canvas internal API.
4. **Complete:** First node finishes. If Claude requested additional nodes (D-02, max 3), repeat steps 1-3 for next node (D-03, sequential).
5. **Done:** All nodes complete. Status bar returns to `AI: idle`.

### Error Recovery

| Error Type | User Feedback | Recovery |
|------------|---------------|----------|
| Network timeout (30s) | Notice: timeout copy. Status bar: error. | Automatic retry with exponential backoff (1s, 2s, 4s). Max 3 retries. |
| API auth error (401) | Notice: `Canvas AI: Invalid API key. Check Settings > Canvas AI.` Status bar: `AI: error`. | No retry. User must fix key. |
| API rate limit (429) | Notice: `Canvas AI: Rate limited. Will retry in {seconds}s.` Status bar: `AI: error`. | Retry after `Retry-After` header value, or 60s default. |
| API server error (500+) | Notice: `Canvas AI: Generation failed. Will retry automatically.` Status bar: `AI: error`. | Exponential backoff, max 3 retries. |
| Stream interrupted | No notice (partial content preserved in node). Status bar: `AI: idle`. | No retry for partial success. Node keeps whatever content was received. |
| Budget exceeded | Persistent notice with settings link. Status bar: `AI: budget`. | Override toggle in settings. |
