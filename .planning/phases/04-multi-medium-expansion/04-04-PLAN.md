<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Manual verification in live Obsidian</name>
  <files>main.js, manifest.json, styles.css</files>
  <action>
    Present the manual verification checklist to the user. The user will copy the built plugin files into their Obsidian vault and test all medium types. This is a human-verify checkpoint -- the user performs the verification and reports results.

    **Setup:**
    1. Copy the built plugin files (`main.js`, `manifest.json`, `styles.css`) to your Obsidian vault's `.obsidian/plugins/canvas-ai/` directory
    2. Reload Obsidian (Cmd+R)
    3. Enable the Canvas AI plugin in Settings > Community Plugins
    4. Verify Claude API key and Runware API key are set in Settings > Canvas AI
    5. Verify "Image save location" field appears in settings with default `canvas-ai-images`

    **Test 1: Text generation (baseline)**
    1. Open a canvas and create a text node with content (e.g., "What are the key principles of distributed systems?")
    2. Click out of the node, wait ~3 seconds
    3. Expected: A new text node appears with relevant markdown content, pulsing animation during streaming

    **Test 2: Code generation (MMED-02)**
    1. Create a text node with technical content (e.g., "Write a function that implements binary search in TypeScript")
    2. Click out, wait ~3 seconds
    3. Expected: Code node appears with fenced code block and syntax highlighting, streams progressively

    **Test 3: Mermaid diagram (MMED-03, MMED-04)**
    1. Create a text node with relationship content (e.g., "The authentication flow: user enters credentials, server validates, issues JWT, client stores token")
    2. Click out, wait ~3 seconds
    3. Expected: Mermaid node shows pulsing while buffering, then "pops" into a complete rendered diagram

    **Test 4: Image generation (MMED-05, MMED-06, MMED-07, MMED-08)**
    1. Create a text node with visual content (e.g., "A vast galaxy with swirling nebulae and distant stars")
    2. Click out, wait ~3 seconds
    3. Expected: Square placeholder (512x512) with pulsing + "Generating image..." text, resolves to file node with image

    **Test 5: Mixed medium + one-per-type (D-01, D-02, D-03)**
    1. Create complex text node triggering multiple mediums
    2. Expected: Multiple distinct types appear (max 1 per type, max 4 total), no overlapping

    **Test 6: AI node isolation (D-14)**
    1. Click on an AI-generated node
    2. Expected: No new generation triggers
  </action>

    - User confirms image generation works (placeholder + file node swap)
